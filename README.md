# Репозиторий сайта ЦМ Крафта

## Руководство

### Запуск

1. Поставьте [Node.js](https://nodejs.org/en/download).

2. Установите зависимости. Для этого откройте в терминале папку с проектом (ту, где лежит package.json) и напишите `npm install`.

3. Запустите проект, написав `npm run start`. После этого веб-сервер будет доступен на `localhost:3000`. 

*Для автоматизации (автозапуск после ребута, запись логов, статус, лёгкий перезапуск и т.д.) см. менеджер процессов [PM2](https://github.com/Unitech/pm2). Команда `pm2 start npm --name "cmc" -- run dev` запустит сервер в виде сервиса, но туториал всё равно почитайте.*

#### Режим разработчика

Чтобы при разработке не перезапускать сервер после каждого изменения, вместо шага 3 введите `npm run dev`. Это запустит службу, которая будет перезапускать сервер сама.

### Работа с правилами

Правила хранятся в виде Markdown-файлов (папка /public/rules). Сайт парсит эти файлы в HTML-код. В самих файлах при желании тоже можно использовать HTML.

Парсинг занимает довольно ощутимое время (около 20мс), поэтому ради экономии ресурсов и сокращения времени отклика сервера правила парсятся в момент запуска и хранятся в памяти. **Поэтому при изменении правил надо перезапустить сервер.**

#### Формат

Файл с основными правилами `general.md` — особенный. Для того, чтобы по нему работал поиск, я придумал специальный формат для разметки правил:

```md
1. **Строительство лаг-машин.**
   <tags>тех</tags>

   Наказание: вечный бан.

2. **Следующее правило.**

   Всем насрать, наказания нет!

   1. **Вложенное правило.**
   <tags>два, тэга</tags>
   
	  Тут можно написать что угодно, на самом деле. Можно и ничего.

	  *Ещё один опциональный параграф, который как-то дополняет правило.*
```

Приведённая структура разобъётся в браузере на следующие блоки:
```JSON
[
	{
		"number": "1",
		"name": "<p><strong>Строительство лаг-машин.</strong></p>",
		"description": "<p>Наказание: вечный бан</p>",
		"tags": ["тех"]
	},
	{
		"number": "2",
		"name": "<p><strong>Следующее правило.</strong></p>",
		"description": "<p>Всем насрать, наказания нет!</p>",
		"tags": []
	},
	{
		"number": "2.1",
		"name": "<p><strong>Вложенное правило.</strong></p>",
		"description": "<p>Тут можно написать что угодно, на самом деле. Можно и ничего.</p> <p><em>Ещё один опциональный параграф, который как-то дополняет правило.</em></p>",
		"tags": ["два", "тэга"]
	},
]
```

Характеристики каждого поля:

| Название      | Описание                                                  | Ограничения                                                                                                         |
|---------------|-----------------------------------------------------------|---------------------------------------------------------------------------------------------------------------------|
| `number`      | Номер правила. Обязательно.                               | Вложенную нумерацию надо оформлять через простые цифры (1., 2., 3.), а не 1.1., 1.2., 1.3..                         |
| `name`        | То, что правило запрещает. Обязательно если нет описания. | Имя обязательно оформлять в `**жирный текст**`                                                                      |
| `description` | Наказание, уточнения. Обязательно если нет названия.      | В описании нельзя использовать **жирный текст** через двойные звёздочки. Если он очень нужен, используйте `<b></b>` |
| `tags`        | Тэги правила. Опционально.                                | Тэги надо писать внутри `<tags></tags>` через запятую и с маленькой буквы                                            |
